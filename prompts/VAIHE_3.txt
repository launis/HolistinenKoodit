Tiedosto: 3_argumentaatioanalyysi.json

TypeScript


interface ArgumentaatioAnalyysi extends BaseJSON {
  toulmin_analyysi: Array<{
    vaite_id: string;
    claim: string;
    data: string;
    warrant: string;              // Oikeutus
    backing: string;              // Tuki (jos on)
  }>;
  kognitiivinen_taso: {
    bloom_taso: string;           // esim. "Analyze" tai "Evaluate"
    strateginen_syvyys: string;   // Arvio onko kyseessä syvällinen vai pinnallinen taso
  };
  walton_skeema: {
    tunnistettu_skeema: string;   // esim. "Argument from Expert Opinion"
    kriittiset_kysymykset: string[]; // Lista kysymyksistä vaiheelle 4
  };
}
}
KÄSKE: Sinun TÄYTYY noudattaa seuraavaa analyysiprotokollaa (Luku 2.4.2):
1.  **Toulmin-dekonstruktio**: Pura argumentti osiin: Väite (Claim), Perusteet (Data), Oikeutus (Warrant), Tuki (Backing), Varauma (Rebuttal) ja Täsmennin (Qualifier).
2.  **Oikeutuksen (Warrant) Arviointi**: Arvioi Oikeutuksen kognitiivinen taso Bloomin taksonomialla (Anderson & Krathwohl 2001). Onko se vain muistamista vai uutta luovaa synteesiä?
3.  **Waltonin Skeema**: Tunnista argumentin taustalla oleva skeema (esim. Argument from Expert Opinion, Analogy, Consequences) (Walton ym. 2008).
4.  **Riskienhallinta**: Kirjaa `metodologinen_loki` -kenttään riski skeeman väärintunnistuksesta ja Toulminin mallin lineaarisuudesta (Crusius & Channell 2003).
