Tiedosto: 6_performatiivisuus_auditointi.json

TypeScript


interface PerformatiivisuusAuditointi extends BaseJSON {
  performatiivisuus_heuristiikat: Array<{
    heuristiikka: string;         // esim. "Epäuskottava lineaarisuus"
    lippu_nostettu: boolean;      // Tunnistettiinko ongelma?
    kuvaus: string;
  }>;
  pre_mortem_analyysi: {
    suoritettu: boolean;
    hiljaiset_signaalit: string[]; // Lista signaaleista, jos löytyi
  };
  yleisarvio_aitoudesta: "Orgaaninen" | "Performatiivinen" | "Epäilyttävä";
}
}
KÄSKE: Sinun TÄYTYY noudattaa seuraavaa performatiivisuuden tunnistusprotokollaa (Luku 2.4.3):
1.  **Heuristiikka-analyysi**: Etsi aktiivisesti seuraavia merkkejä:
    *   **Epäuskottava lineaarisuus**: Onko prosessi liian suoraviivainen ja virheetön?
    *   **Pinnallinen vuorovaikutus**: Onko kognitiivinen syvyys vähäistä?
    *   **Kognitiivinen epäsuhta**: Ristiriita reflektion ja historian välillä.
    *   **Keinotekoinen monimutkaisuus**: Turhia vaiheita näyttävyyden vuoksi.
    *   **Matriisin optimointi**: Reflektio vastaa epäilyttävän tarkasti kriteereitä.
    *   **Itsetehostus**: Liiallinen oman erinomaisuuden korostaminen.
2.  **Pre-Mortem Analyysi**: Oleta, että reflektio on väärennös. Etsi todisteita, jotka tukevat tätä oletusta.
3.  **Epäilyttävä Täydellisyys**: Jos suoritus on täydellinen ilman näkyvää kitkaa, liputa se automaattisesti.

KÄSKE: (Esimerkki oikeasta rakenteesta):
  },
  "yleisarvio_aitoudesta": "Orgaaninen"
}
ÄLÄ käytä avaimia kuten "pisteytys", "taso" tai "kriteeri1" ellei niitä ole yllä mainittu. Käytä vain yllä olevia.
